网关与 AMAP_API_URL 配置说明

常见误解：不需要把你的网站挂到 *.amap.com 子域名。/api/amap 是你自己站点上的 HTTP 路径，由部署平台（Functions / 网关 / Nginx）把请求转给服务端代码；不是高德服务器上的目录，也不是 地图 JS API 2.0 概述 里那种「在浏览器里加载 webapi.amap.com 画地图」的同一件事。

若直觉「同域访问不了 /api/amap」

若你当前是纯静态托管（例如本机 python -m http.server、只上传静态文件到 OSS/CDN、没有配置任何「无服务器函数 / 反向代理」），则浏览器访问 /api/amap 会 404 —— 因为没有任何程序在监听这个路径。此时要么：① 把整站（含 edge-functions）部署到支持 Pages Functions / 类似能力的平台并配置 AMAP_REST_KEY；② 在 AMAP_API_URL 里填写已经部署好的另一套接口完整地址；③ 用 Nginx 把 /api/amap 反代到你的云函数。

1. 本项目的 /api/amap（与高德 /_AMapService 无关）

这是你自己部署的 HTTP 接口：浏览器 POST JSON（如 action: resolvePlaces），服务端用 AMAP_REST_KEY 访问 https://restapi.amap.com。仓库内实现为 edge-functions/api/amap.js，映射路径一般为 /api/amap。

2. EdgeOne / Pages Functions

将静态站点与 Functions 同域部署时，index.html 里 AMAP_API_URL 可留空，前端使用相对路径 /api/amap。环境变量中配置 AMAP_REST_KEY。

3. 腾讯云 API 网关 + 云函数

  • 新增路由：POST /api/amap（或自定义路径，则须把 AMAP_API_URL 设为完整 URL）。
  • 开启 CORS：OPTIONS 预检、响应头 Access-Control-Allow-Origin（可为静态页来源或 *）。
  • 502/404 的错误响应也需带 CORS 头，否则浏览器只显示「Failed to fetch」。

4. globalThis.__APP_CONFIG__.AMAP_API_URL

  • 留空：请求同域 /api/amap。
  • https://api.example.com：自动请求 https://api.example.com/api/amap。
  • 已含路径的绝对地址（如网关发布的完整 URL）：按原样使用，不自动追加。
  • *.tencentscf.com 且 URL 无路径：视为函数根 URL，不追加 /api/amap（若云函数未实现本接口，请改为带路径的完整 URL 或单独部署 amap 函数）。

5. 高德官方：JS API 的 /_AMapService 代理（地图瓦片 / Web 服务走代理）

这是给 _AMapSecurityConfig.serviceHost 用的另一套 Nginx 规则，用于把浏览器对高德的请求经你的域名转发并附带 jscode。示例（摘自官方文档，请把「你的安全密钥」换成控制台里的值）： 

location /_AMapService/ {
  set $args "$args&jscode=你的安全密钥";
  proxy_pass https://restapi.amap.com/;
}

然后在「设置」中填写 JS API 代理 serviceHost 为 https://你的域名/_AMapService。官方文档： JS API 安全密钥使用。
